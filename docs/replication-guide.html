1. Project Overview

What We Built

A complete 12-week PhD course website featuring:

  • 12 Week Pages - Each with learning objectives, topics, required readings, exercises
  • PDF Slide Decks - 12 Beamer presentations with charts
  • Interactive Quizzes - 60+ questions with immediate feedback
  • Jupyter Notebooks - 15+ Colab-linked notebooks
  • Reading Guides - Structured paper reading assignments
  • Exercises - Graded coding assignments
  • Python Playground - Browser-based Pyodide environment

2. Architecture

Repository Structure

The project uses a monorepo structure with the website source in /docs:

Key Insight GitHub Pages can deploy from: (1) root /, (2) /docs folder, or (3) gh-pages branch. We use /docs to keep website separate from lecture source files.

Data Flow

YAML Data Files          Liquid Templates         HTML Output
----------------         ----------------         -----------
_data/weeks.yml    -->   _layouts/week.html  -->  weeks/week-1.html
_data/readings.yml -->   _layouts/week.html  -->  (reading section)
_data/exercises.yml -->  _layouts/week.html  -->  (exercise section)
_data/quizzes.yml  -->   _includes/quiz.html -->  (quiz component)

9. Deployment

GitHub Actions Workflow

# .github/workflows/pages.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: ["main"]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Ruby
        uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.2'
          bundler-cache: true
          working-directory: docs

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Build with Jekyll
        run: |
          cd docs
          bundle exec jekyll build --baseurl "${{ steps.pages.outputs.base_path }}"
        env:
          JEKYLL_ENV: production

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: docs/_site

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

Deployment Steps

1
Enable GitHub Pages

Go to Settings > Pages > Build and deployment > Source: GitHub Actions

2
Make Repository Public 
gh repo edit --visibility public --accept-visibility-change-consequences

GitHub Pages requires public repos (or GitHub Pro for private).

3
Enable Discussions (Optional) 
gh repo edit --enable-discussions

Required if your pages link to GitHub Discussions.

4
Push to Main 
git add .
git commit -m "Deploy course website"
git push origin main

GitHub Actions will automatically build and deploy.

5
Check Deployment Status 
gh run list --limit 5
gh run view --log-failed  # If failed

Debugging Failed Builds

# View recent workflow runs
gh run list --limit 5

# View failed run logs
gh run view <run-id> --log-failed

10. Common Errors and Fixes

Error Reference Table

Error Cause Fix
404 Not Found on PDFs Repository is private gh repo edit --visibility public
404 Not Found on PDFs (public repo) PDFs excluded in _config.yml or wrong path Remove *.pdf from exclude list; use /slides/ path
Liquid error: Nesting too deep Liquid tag in HTML comment Remove {% %} from comments
264px and 50rem have incompatible units Mixing px with rem in Sass math Convert px to rem: 264px -> 16.5rem
Navigation shows /week-1 instead of /week-0 Liquid filter chain order Pre-calculate numbers before append
GitHub Discussions 404 Discussions not enabled gh repo edit --enable-discussions
Colab badge broken Private repo or wrong path Make public; verify notebook path
No such file or directory in build Include file doesn't exist Verify _includes/ files exist
Styles not applying Color scheme not set Add color_scheme: custom to _config.yml
Search not working Search plugin not configured Enable search_enabled: true

Detailed Fix: Liquid Filter Chain

The Problem 
{{ '/weeks/week-' | append: week_num | minus: 1 | relative_url }}

When week_num = 2, this produces /weeks/week-1 (correct by accident).

But the math is wrong! append creates string "/weeks/week-2", then minus: 1 tries to subtract from a string.

The Fix 
{% assign prev_week_num = week_num | minus: 1 %}
{% assign next_week_num = week_num | plus: 1 %}
{{ '/weeks/week-' | append: prev_week_num | relative_url }}

Always do math BEFORE string operations.

Detailed Fix: Sass Units

The Problem 
// Your custom.scss
$nav-width: 264px;

// Just the Docs theme (internally)
$something: $nav-width + 50rem;  // ERROR!
The Fix 
$nav-width: 16.5rem;  // 264 / 16 = 16.5

Always use rem units when customizing Just the Docs theme.
