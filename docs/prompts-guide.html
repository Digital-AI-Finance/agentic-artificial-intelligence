Phase 1: Repository Setup

1 Create GitHub Repository

Create a new GitHub repository for my course: - Organization: [ORG_NAME] - Repository name: [REPO_NAME] - Description: "[COURSE_TITLE] - A 12-week PhD/graduate course" - Make it private initially - Initialize with README Use gh CLI to create it.
Repository created at github.com/[ORG_NAME]/[REPO_NAME]

2 Create Directory Structure

Create the following directory structure for my course repository at [LOCAL_PATH]: Root level: - L01_[Week1_Topic]/ through L12_[Week12_Topic]/ (lecture folders) - docs/ (Jekyll website) - .github/workflows/ (CI/CD) - README.md - status.md - changelog.md Inside docs/: - _config.yml - _data/ (weeks.yml, readings.yml, exercises.yml, quizzes.yml) - _layouts/ (week.html) - _includes/ (quiz.html) - _sass/color_schemes/ (custom.scss) - weeks/ (index.md, week-1.md through week-12.md) - slides/ (for PDF files) - index.md - slides.md - Gemfile Create all folders and placeholder files.

3 Initialize Jekyll Configuration

Create the Jekyll _config.yml for my course website with: - Title: "[COURSE_TITLE]" - Base URL: "/[REPO_NAME]" - URL: "https://[ORG_NAME].github.io" - Theme: just-the-docs v0.8.2 (remote theme) - Color scheme: custom - Search enabled - GitHub edit links enabled - Course metadata: - Code: [COURSE_CODE] - Semester: [SEMESTER] - Instructor: [INSTRUCTOR_NAME] - Institution: [INSTITUTION] IMPORTANT: Do NOT exclude *.pdf files. Write the complete _config.yml file.

4 Create Gemfile

Create the Gemfile for Jekyll with: - jekyll ~> 4.3 - just-the-docs ~> 0.8.2 - Plugins: jekyll-seo-tag, jekyll-relative-links, jekyll-github-metadata - Windows compatibility gems Write to docs/Gemfile.

5 Create GitHub Actions Workflow

Create a GitHub Actions workflow for deploying Jekyll to GitHub Pages: - Trigger on push to main - Use Ruby 3.2 - Build from docs/ folder - Deploy to GitHub Pages using actions/deploy-pages Write to .github/workflows/pages.yml
Phase 2: Website Structure

6 Define Course Weeks Data

Create the weeks.yml data file for my 12-week course on [COURSE_TITLE]. For each week, I need: - number (1-12) - title - folder (lecture folder name, e.g., L01_Topic_Name) - description (1-2 sentences) - topics (4-5 bullet points) - learning_objectives (3-4 objectives) - papers (2-3 required readings with title, authors, year, arxiv ID or URL) - notebooks (list of Jupyter notebooks with name and file path) Here are my 12 weeks: Week 1: [Topic 1] Week 2: [Topic 2] Week 3: [Topic 3] Week 4: [Topic 4] Week 5: [Topic 5] Week 6: [Topic 6] Week 7: [Topic 7] Week 8: [Topic 8] Week 9: [Topic 9] Week 10: [Topic 10] Week 11: [Topic 11] Week 12: [Topic 12] Generate comprehensive content for each week. Research real papers on arXiv for each topic. Write to docs/_data/weeks.yml.
Tip: Be specific about your topics. The more detail you provide, the better the generated content.

7 Create Reading Guides Data

Create docs/_data/readings.yml with reading guides for all 12 weeks. For each week include: - week number - title (reading guide title) - file (path to markdown file) - estimated_time (e.g., "2-3 hours") - topics (3-4 key topics) - description - primary_paper (title, authors, year, venue, arxiv) - secondary_papers (2-3 additional papers) Base this on the weeks.yml content. Ensure all arXiv IDs are real and verified.

8 Create Exercises Data

Create docs/_data/exercises.yml with exercises for all 12 weeks. For each week include: - week number - title - file (path to markdown file) - total_points (e.g., 100) - estimated_time (e.g., "4-6 hours") - difficulty (Beginner/Intermediate/Advanced) - description - objectives (3-4 learning objectives) - tasks (3-4 tasks with name, points, description) Make exercises progressively more challenging. Week 1 should be beginner-friendly, Week 12 should be advanced.

9 Create Week Layout Template

Create the week layout template at docs/_layouts/week.html that: 1. Gets week data from site.data.weeks based on page.week_number 2. Shows navigation to previous/next weeks 3. Displays: - Week title and description - Progress bar (week X of 12) - Learning Objectives (as HTML list) - Topics Covered (as HTML list) - Resources section with PDF download button - Jupyter Notebooks with Colab badges - Required Readings as HTML table - Reading Guide card (from readings.yml) - Exercise card (from exercises.yml) - Discussion & Questions section with GitHub links CRITICAL: Use HTML tags (h2, h3, ul, li, table) NOT markdown syntax (##, -, |). CRITICAL: Pre-calculate prev_week_num and next_week_num BEFORE using in append filter. Base the styling on the course color scheme.

10 Create Quiz Component

Create an interactive quiz component at docs/_includes/quiz.html that: 1. Takes 'id' and 'questions' parameters 2. Shows one question at a time with multiple choice options 3. Provides immediate feedback (correct/incorrect with explanation) 4. Tracks progress with a progress bar 5. Shows final score at the end 6. Has a "Try Again" button to restart Include all CSS and JavaScript inline. CRITICAL: Do NOT put any Liquid tags ({% %}) inside HTML comments - Jekyll processes them anyway.

11 Create Custom Styling

Create custom SCSS styling at docs/_sass/color_schemes/custom.scss with: Color palette: - Primary: #3333B2 (deep purple-blue) - Secondary: #0066CC (bright blue) - Accent: #FF7F0E (orange) - Success: #2CA02C (green) - Danger: #D62728 (red) Include styles for: - Progress bars - Cards (week, reading, exercise) - Buttons (primary, secondary, outline) - Resource links - Badges - Dark mode support CRITICAL: Use rem units NOT px for any values that will be used in calculations (e.g., $nav-width: 16.5rem NOT 264px).

12 Create Week Pages

Create all 12 week markdown files in docs/weeks/: Each file (week-1.md through week-12.md) needs: - Frontmatter with layout: week, title, week_number, nav_order, parent: Weeks - Optional supplementary content sections: - Exercise (hands-on task description) - Discussion Questions (3 thought-provoking questions) - Additional Resources (3-5 relevant links) Also create docs/weeks/index.md as the parent page listing all weeks.

13 Create Homepage

Create docs/index.md as the course homepage with: 1. Course title and welcome message 2. Quick overview table showing all 12 weeks with links 3. Course information (instructor, semester, credits) 4. Getting started section 5. Links to slides, syllabus, resources Keep it compact - use a table layout for the week listing rather than large cards.

14 Create Slides Page

Create docs/slides.md as a download page for all slide decks: 1. Title: "Lecture Slides" 2. Table with all 12 weeks showing: - Week number - Topic - Download link (PDF) 3. Links should use: {{ '/slides/FOLDER_NAME.pdf' | relative_url }} Also create the docs/slides/ folder where PDFs will be stored.
Phase 3: Course Content Generation

15 Generate Week 1 Content

Generate complete content for Week 1: [Week 1 Topic] Create: 1. Detailed learning objectives (4-5 specific, measurable objectives) 2. Topic outline (main concepts to cover) 3. Required readings (find 2-3 real papers on arXiv) 4. Reading guide markdown file with: - Paper summary - Key concepts to understand - Discussion questions 5. Exercise markdown file with: - Clear instructions - Starter code or templates - Grading rubric 6. Quiz questions (5-6 multiple choice with explanations) Research actual papers and resources. Do not hallucinate citations.
Tip: Repeat this prompt for weeks 2-12, replacing the topic each time. You can batch multiple weeks: "Generate content for Weeks 2, 3, and 4: [Topic 2], [Topic 3], [Topic 4]"

16 Create Jupyter Notebooks

Create a Jupyter notebook for Week [N]: [Topic] The notebook should: 1. Have a clear title and introduction 2. Include necessary imports 3. Provide step-by-step explanations 4. Include runnable code cells with examples 5. Have exercises for students to complete 6. Work in Google Colab (use !pip install for dependencies) Save to L[N]_[Topic_Folder]/notebooks/L[N]_[notebook_name].ipynb Make the notebook self-contained and educational.
Phase 4: Beamer Slides Creation

17 Create Slide Deck Structure

Create a Beamer slide deck for Week [N]: [Topic] Structure: - 20-30 slides - Madrid theme, 8pt font - Title slide with course info - Outline slide - Content slides organized by subtopic - Summary slide - References slide For charts: - Create one standalone Python script per chart - Each chart in its own subfolder (01_chart_name/, 02_chart_name/) - figsize=(10, 6) for all charts - Minimum font size 14pt (scaled for 70% display) - Save as chart.pdf Include 3-5 charts per lecture. Topics for charts: [List specific visualizations needed for this topic] Write the .tex file to L[N]_[Topic]/L[N]_[Topic].tex

18 Generate Charts

Create the chart Python scripts for Week [N]: [Topic] For each chart: 1. Create folder: L[N]_[Topic]/[NN]_[chart_name]/ 2. Create chart.py with: - Descriptive docstring - matplotlib with consistent style - rcParams for font sizes (min 14pt) - Color palette: MLPURPLE=#3333B2, MLBLUE=#0066CC, MLORANGE=#FF7F0E - figsize=(10, 6) - Save to chart.pdf 3. Run the script to generate chart.pdf Charts needed: 1. [Chart 1 description] 2. [Chart 2 description] 3. [Chart 3 description] Use synthetic/simulated data. Make charts educational and clear.

19 Compile Slides

Compile the Beamer slides for Week [N]: 1. Run all chart.py scripts in L[N]_[Topic]/ subfolders 2. Compile L[N]_[Topic].tex with pdflatex (run twice for references) 3. Check for "Overfull" warnings - if any, fix by: - Reducing bullet points - Using smaller chart width (0.55 instead of 0.65) - Splitting content across slides 4. Copy final PDF to docs/slides/L[N]_[Topic].pdf 5. Move auxiliary files (.aux, .log, .nav, etc.) to temp/ folder Verify zero overflow warnings before finalizing.
Warning: Never allow "Overfull \vbox" or "Overfull \hbox" warnings. These cause content to overflow off slides.
Phase 5: Deployment

20 Make Repository Public

Make the repository public and enable GitHub Pages: 1. Make repo public: gh repo edit --visibility public --accept-visibility-change-consequences 2. Enable GitHub Discussions: gh repo edit --enable-discussions 3. Verify GitHub Pages is set to deploy from GitHub Actions 4. Push all changes to main branch 5. Wait for deployment and verify site loads Open the deployed site: https://[ORG_NAME].github.io/[REPO_NAME]/

21 Verify Deployment

Take a screenshot of the deployed site and verify: 1. Homepage loads correctly 2. All 12 weeks are accessible 3. Navigation between weeks works 4. PDF slides download correctly 5. Colab notebook links work 6. Required readings table renders properly 7. Exercise and reading guide cards display correctly Use Playwright to automate verification across all pages.

22 Run Link Checker

Run a comprehensive link check on the deployed site: 1. Check all internal links (week pages, slides, resources) 2. Check all external links (arXiv, GitHub, documentation sites) 3. Report any 404 errors or broken links 4. Fix broken links in source files 5. Re-deploy and verify fixes Generate a report of all links checked and their status.
Phase 6: Troubleshooting

23 Fix Common Issues

Debug and fix any issues with the deployed site. Common issues to check: 1. Raw markdown showing (## headers, | tables) - convert to HTML in layouts 2. PDF 404 errors - check paths and _config.yml exclude list 3. Sass compilation errors - use rem not px for calculations 4. Navigation showing wrong URLs - pre-calculate week numbers before append 5. Infinite recursion - remove Liquid tags from HTML comments 6. GitHub Discussions 404 - enable discussions on repo For each issue found: 1. Identify the root cause 2. Fix the source file 3. Commit with descriptive message 4. Verify fix on deployed site Document all fixes in a JSON file.

24 Final Verification

Perform final verification of all 12 weeks: 1. Take screenshots of each week page 2. Verify all sections render correctly: - Learning Objectives - Topics Covered - Resources (PDF download) - Jupyter Notebooks (Colab badges) - Required Readings (table) - Reading Guide (card) - Exercise (card) - Discussion section 3. Check mobile responsiveness 4. Verify search functionality 5. Test dark mode if enabled Create a verification report JSON with status for each week.
