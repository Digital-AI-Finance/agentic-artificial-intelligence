# Contributing to Agentic AI Course

Thank you for your interest in contributing! This course is an open educational resource, and we welcome contributions from the community.

## Ways to Contribute

### 1. Report Issues

Found an error in the slides? A broken link? Incorrect information?

- [Report a bug](https://github.com/Digital-AI-Finance/agentic-artificial-intelligence/issues/new?template=bug_report.md)
- [Suggest an improvement](https://github.com/Digital-AI-Finance/agentic-artificial-intelligence/issues/new?template=feature_request.md)

### 2. Fix Typos and Errors

Small fixes are always welcome:

1. Fork the repository
2. Make your changes
3. Submit a pull request

### 3. Improve Documentation

Help make the course more accessible:

- Clarify confusing explanations
- Add examples
- Improve code comments
- Translate content

### 4. Add Content

Contribute new material:

- New exercises
- Additional reading guides
- Code examples
- Quiz questions

### 5. Review Content

Help maintain quality:

- Verify citations and references
- Check code examples work
- Validate claims and statistics
- Test notebooks

## Contribution Guidelines

### Code Style

**Python:**
- Follow PEP 8
- Use type hints where possible
- Include docstrings for functions
- Keep examples simple and focused

**LaTeX/Beamer:**
- 8pt font size
- Madrid theme
- No TikZ for charts (use Python)
- Maximum 4 bullet points per slide

### Pull Request Process

1. **Fork** the repository
2. **Create a branch** for your changes
   ```bash
   git checkout -b fix/typo-in-week3
   ```
3. **Make your changes**
4. **Test** your changes locally
5. **Commit** with a clear message
   ```bash
   git commit -m "Fix typo in L03 slides"
   ```
6. **Push** to your fork
7. **Open a PR** with a description of changes

### Commit Message Format

```
<type>: <short description>

<optional longer description>
```

Types:
- `fix`: Bug fixes
- `feat`: New features
- `docs`: Documentation changes
- `style`: Formatting changes
- `refactor`: Code restructuring
- `test`: Adding tests
- `chore`: Maintenance tasks

### Content Standards

#### Accuracy
- All claims must be verifiable
- Include citations for statistics
- Link to primary sources when possible

#### Accessibility
- Use descriptive alt text for images
- Ensure proper heading hierarchy
- Test with screen readers

#### Consistency
- Follow existing formatting
- Match the tone and style
- Use consistent terminology

## Review Process

1. A maintainer will review your PR
2. They may request changes
3. Once approved, it will be merged
4. Your contribution will be credited

## Code of Conduct

- Be respectful and constructive
- Focus on the content, not the person
- Welcome newcomers
- Assume good intentions

## Getting Help

- Ask in [GitHub Discussions](https://github.com/Digital-AI-Finance/agentic-artificial-intelligence/discussions)
- Tag issues appropriately
- Be specific about problems

## Recognition

Contributors are recognized in:
- Pull request credits
- Release notes
- The README contributors section

Thank you for helping improve this course!
